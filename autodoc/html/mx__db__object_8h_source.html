19/*
20 * MODULE/CLASS : mx_db_object
21 *
22 * AUTHOR : Andrew Haisley
23 *
24 *
25 *
26 * DESCRIPTION:
27 *
28 * mx_db_object provides database access functions for documents
29 * and all objects contained within documents.
30 *
31 * NOTES:
32 *
33 * Things are arranged so that you only need worry about your local
34 * memory management. Any objects/values that you pass into this
35 * object get copied (if necessary) and any that come out are
36 * stand-alone copies. As long as you delete your own objects
37 * and any that you get back as a result of calling methods on this
38 * object you'll be fine.
39 *
40 * There are also methods that return read-only copies of things -
41 * you should use these when accessing large attributes that
42 * you don't want to modify (e.g. drawing a raster) as they don't
43 * do any copying.
44 *
45 *
46 *
47 */
